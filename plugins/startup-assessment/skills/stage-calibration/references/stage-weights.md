## Table 3: Weight Redistribution Formula

When a domain is deactivated or weights are adjusted:

```
Step 1: Sum remaining active domain weights
sum_remaining = Σ(weight of all active domains after deactivation)

Step 2: Calculate redistribution ratio
redistribution_ratio = 1.0 / sum_remaining

Step 3: Apply to all remaining domains
new_weight[domain_i] = old_weight[domain_i] × redistribution_ratio

Step 4: Verify total
Σ(all new_weight) must equal 1.0 (tolerance: ±0.01)
```

### Example: Pre-Seed to Seed Transition

**Pre-Seed active domains and weights:**
- D1: 0.22, D2: 0.20, D3: 0.08, D5: 0.08, D6: 0.28, D7: 0.04, D8: 0.02, D9: 0.01, D10: 0.01

**Activation event:** D4 (Go-to-Market) elevated from inactive to secondary

**New target distribution:** Seed configuration with D4 at 10%

**Redistribution:**
- D1: 0.22 → 0.18 (−0.04, 9% reduction)
- D2: 0.20 → 0.15 (−0.05, 25% reduction)
- D3: 0.08 → 0.12 (+0.04, 50% increase)
- D4: 0.00 → 0.10 (+0.10, new primary)
- D5: 0.08 → 0.15 (+0.07, 87% increase)
- D6: 0.28 → 0.22 (−0.06, 21% reduction)
- D7: 0.04 → 0.05 (+0.01, 25% increase)
- D8: 0.02 → 0.02 (stable)
- D9: 0.01 → 0.01 (stable)
- D10: 0.01 → 0.00 (deactivated)

**Total verification:** 0.18+0.15+0.12+0.10+0.15+0.22+0.05+0.02+0.01 = 1.00 ✓

## Table 4: Hard Blocker Modules by Stage

### Hard Blocker Definition
A hard blocker module is one where a completeness score below the threshold **automatically results in a "no-go" recommendation** for that domain, regardless of other module scores in the domain.

### Pre-Seed Hard Blockers
| Module | Threshold | Consequence |
|--------|-----------|-------------|
| Problem Validation (D1) | ≥0.6 | If below: No evidence problem is real; assessor should request customer conversation evidence |
| Founder Background (D6) | ≥0.6 | If below: No relevant expertise; assessor should request domain background or advisor substitution |
| Customer Conversations (D5) | ≥0.5 | If below: No market contact; assessor should request evidence of customer engagement |

### Seed Hard Blockers
| Module | Threshold | Consequence |
|--------|-----------|-------------|
| Problem Validation (D1) | ≥0.6 | If below: Market need unvalidated |
| Revenue Evidence (D5) | ≥0.5 | If below: No commercial proof; request customer evidence or LOI |
| Customer Count and Engagement (D5) | ≥0.6 | If below: No adoption signal; request active user/paying customer evidence |
| Founder Background (D6) | ≥0.6 | If below: No relevant leadership experience |

### Series A Hard Blockers
| Module | Threshold | Consequence |
|--------|-----------|-------------|
| Unit Economics (D3) | ≥0.7 | If below: PMF unproven; request LTV/CAC analysis or cohort metrics |
| Revenue Consistency (D5) | ≥0.7 | If below: Revenue not repeatable; request monthly revenue trend |
| Customer Retention (D5) | ≥0.6 | If below: Unit economics sustainability unknown; request churn rate |
| Financial Model Quality (D7) | ≥0.7 | If below: Projections not driver-based; request rebuild with CAC/LTV/churn inputs |

### Series B Hard Blockers
| Module | Threshold | Consequence |
|--------|-----------|-------------|
| Unit Economics (D3) | ≥0.8 | If below: Scale economics not understood; requires detailed cohort LTV analysis |
| Historical Financial Performance (D7) | ≥0.7 | If below: Financial transparency insufficient; request audited or management accounts |
| Revenue and Growth Rate (D5) | ≥0.7 | If below: Growth trajectory unclear; request historical monthly revenue |
| Customer Retention and Expansion (D5) | ≥0.7 | If below: Cohort quality unproven; request detailed retention and expansion cohort analysis |

## Table 5: Stage Misposition Indicators

If a submission exhibits these signals, reassess upward or downward as indicated:

### Indicators to Reassess Downward (to Earlier Stage)

| Signal | Assessment | Recommendation |
|--------|-----------|---|
| Claims Series A stage but has <$5K MRR (SaaS) | Stage inflation | Reassess as Seed |
| Claims Seed but team is solo founder with no customers | Stage inflation | Reassess as Pre-Seed |
| Raises Series B in name but revenue is <$500K ARR | Stage inflation | Reassess as Series A |
| No audited accounts available at Series B | Process gap | Request or reassess as late Series A |
| Revenue is 100% single customer | Traction misidentification | Flag customer concentration; may affect stage confidence |

### Indicators to Reassess Upward (to Later Stage)

| Signal | Assessment | Recommendation |
|--------|-----------|---|
| Claims Pre-Seed but has $50K MRR and paying customers | Stage mislabeling | Reassess as Seed |
| Claims Seed but has $500K+ ARR with 50+ customers | Stage mislabeling | Reassess as Series A |
| Series A submission with mature $10M ARR | Stage mislabeling | Reassess as Series B |
| Earlier stage submission with audited accounts and cohort analysis | Stage mislabeling | Reassess upward |

## Table 6: Debt Instrument Adjustments

When assessing a debt instrument (convertible note, revenue-based financing, traditional term loan) at any stage:

### Weight Override for Debt Assessment

| Domain | Normal Weight (Series A) | Debt Override | Rationale |
|--------|---------|--------|---------|
| Financial Performance (D7) | 10% | 25% | Repayment capacity is primary |
| Risk Assessment (D8) | 2% | 15% | Default risk must be assessed |
| Capital Structure (D9) | 0% | 8% | Debt terms and covenants critical |
| Traction (D5) | 20% | 15% | Revenue matters but as cash generation, not growth |
| Go-to-Market (D4) | 17% | 10% | GTM scalability less critical than cash runway |
| Business Model (D3) | 18% | 12% | Unit economics matter as cash driver |

**Total reassigned weight: 75% vs. 67% in equity Series A**

### Hard Blockers Added for Debt

| Module | Threshold | Rationale |
|--------|-----------|---------|
| Cash Burn and Runway (D7) | ≥0.7 | If below: Unable to assess repayment capacity |
| Debt Service Coverage Ratio (D7) | ≥0.6 | If below: Repayment plan unclear or insufficient |
| Covenant Compliance (D9) | ≥0.7 | If below: Default risk imminent or terms misunderstood |
| Collateral Quality (D9) | ≥0.5 | If below: Security insufficient for loan size |
